Why This Is Vulnerable

1. Plain Text Password Storage

Passwords are stored in plain text in the database. If the database is compromised, all passwords are immediately exposed.

Fix: Use bcrypt, Argon2, or scrypt to hash passwords with a unique salt.

2. No Password Complexity Requirements

Users can set extremely weak passwords like "123456" or "password".

Fix: Enforce minimum length, complexity requirements (uppercase, lowercase, numbers, special characters).

3. No Rate Limiting

Attackers can make unlimited login attempts without any restrictions.

Fix: Implement rate limiting (e.g., 5 requests per minute per IP).

4. No Account Lockout

Accounts are never locked out, allowing unlimited brute force attempts.

Fix: Lock accounts after a certain number of failed attempts (e.g., 5 attempts).

5. Specific Error Messages

Error messages reveal whether a username exists, enabling username enumeration attacks.

Fix: Use generic error messages like "Invalid credentials".
