SQL Injection Vulnerability

⚠️ WARNING: This implementation is intentionally vulnerable

This demo uses string concatenation to build SQL queries, which makes it vulnerable to SQL injection attacks. Never use this approach in production!

Why is this vulnerable?

The backend code constructs the SQL query by directly concatenating user input into the query string: 

query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{password}'"

An attacker can manipulate the input to change the query's logic. For example, entering ' OR '1'='1 as the password transforms the query into: 

SELECT * FROM users WHERE username = 'admin' AND password = '' OR '1'='1'

Since '1'='1' is always true, the query returns all users, bypassing authentication.
