SQL Injection Protection

✅ This implementation uses secure coding practices

This demo uses parameterized queries (prepared statements) to prevent SQL injection attacks. This is the recommended approach for production.

Why is this secure?

The backend code uses parameterized queries with placeholders (?) instead of string concatenation: 

query = "SELECT * FROM users WHERE username = ? AND password = ?"
cursor.execute(query, (username, password))

The database treats the parameters as data, not as executable code. Even if an attacker enters ' OR '1'='1, it's treated as a literal string value, not as SQL syntax.

This means the query will look for a user with the literal username ' OR '1'='1 (which doesn't exist), rather than executing it as SQL logic.
