Vulnerable vs Secure Implementation

Educational demonstration of OWASP A01: Broken Access Control

This educational application demonstrates how Broken Access Control vulnerabilities work by comparing a vulnerable implementation with a secure implementation.

What is Broken Access Control?
Broken Access Control occurs when users can access resources or perform actions outside their intended permissions. This is currently ranked as A01:2025 in the OWASP Top 10.

Sample Credentials:

User: user / user123
User2: user2 / user123

Vulnerabilities Demonstrated

1. Unauthorized Admin Access:

Regular users can access the admin panel by directly typing the URL

2. IDOR (Insecure Direct Object Reference):

Users can view other users' profiles by changing the ID in the URL

3. Missing Role Checks:

Admin-only actions are visible and executable by all users
