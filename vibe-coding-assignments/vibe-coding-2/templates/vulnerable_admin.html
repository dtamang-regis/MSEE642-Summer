⚠️ CRITICAL VULNERABILITY: No role check on admin endpoint

You are viewing the admin panel. If you are NOT an admin, this demonstrates a severe broken access control vulnerability.

Why is this vulnerable?

The /vulnerable/admin endpoint does not check if the logged-in user has the admin role. It only checks if the user is logged in. This means any logged-in user can access this sensitive admin panel by simply typing the URL. 

# VULNERABLE CODE (in app.py):
@app.route('/vulnerable/admin')
def vulnerable_admin():
    if 'vulnerable_user_id' not in session:
        return redirect(url_for('vulnerable_login'))
    # NO ROLE CHECK HERE!
    return render_template('vulnerable_admin.html', ...)

Secure implementation would check: 

# SECURE CODE:
if session.get('role') != 'admin':
    return render_template('403.html'), 403
