✅ SECURE: Viewing Your Own Profile Only

You are viewing your own profile. The application verified that the requested profile ID belongs to you before granting access.

Why is this secure?

The profile endpoint /secure/profile/<user_id> verifies that the requested user_id belongs to the currently logged-in user. Users cannot view other users' profiles by changing the ID in the URL. 

# SECURE CODE (in app.py):
@app.route('/secure/profile/')
def secure_profile(user_id):
    current_user_id = session.get('secure_user_id')
    
    # CHECK if user_id belongs to current user!
    if current_user_id != user_id:
        return render_template('secure_403.html'), 403
