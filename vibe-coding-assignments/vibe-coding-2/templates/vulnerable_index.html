Broken Access Control

Educational demonstration of access control vulnerabilities

⚠️ WARNING: This implementation is intentionally vulnerable

This demo demonstrates common Broken Access Control vulnerabilities including unauthorized admin access, IDOR (Insecure Direct Object Reference), and missing role-based access control. Never use this approach in production!

What makes this vulnerable?

1. No Role-Based Access Control (RBAC):
The application does not check user roles before granting access to sensitive endpoints. Any logged-in user can access the admin panel by simply typing the URL.

2. IDOR (Insecure Direct Object Reference):
Users can access other users' profiles by changing the ID in the URL. The application doesn't verify if the requested resource belongs to the current user.

3. Missing Authorization Checks:
Admin-only actions are visible and executable by all users, regardless of their role.
