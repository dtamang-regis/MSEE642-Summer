Why is this vulnerable?

The profile endpoint /vulnerable/profile/<user_id> does not verify that the requested user_id belongs to the currently logged-in user. Any user can change the ID in the URL to view other users' profiles. 

# VULNERABLE CODE (in app.py):
@app.route('/vulnerable/profile/')
def vulnerable_profile(user_id):
    # NO CHECK if user_id belongs to current user!
    profile_user = conn.execute('SELECT * FROM users WHERE id = ?', 
                               (user_id,)).fetchone()
    return render_template('vulnerable_profile.html', ...)

Secure implementation would check: 

# SECURE CODE:
current_user_id = session.get('user_id')
if current_user_id != user_id:
    return render_template('403.html'), 403
