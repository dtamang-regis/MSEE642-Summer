Broken Access Control Assignment

List of screenshots needed for assignment submission

This page lists all the screenshots you need to capture for your assignment submission. Take screenshots of each scenario described below to demonstrate your understanding of Broken Access Control vulnerabilities and their fixes.

Required Screenshots

1. Landing Page

Description: The main landing page showing the project overview, explanation of Broken Access Control, and buttons for Vulnerable Demo and Secure Demo.

URL: http://localhost:5002/

What to capture: The complete landing page with both demo options visible.

2. Vulnerable Admin Access

Description: Demonstrate that a regular user can access the admin panel by typing the URL directly in the vulnerable demo.

Steps:

  1. Login as regular user (user / user123) in vulnerable demo
  2. Navigate directly to /vulnerable/admin
  3. Capture the admin panel showing you accessed it as a regular user

What to capture: The vulnerable admin panel showing the regular user's username and role, demonstrating unauthorized access.

3. Vulnerable Profile Access (IDOR)

Description: Demonstrate the IDOR vulnerability where a regular user can view another user's profile by changing the ID in the URL.

Steps:

  1. Login as regular user (user / user123) in vulnerable demo
  2. Navigate to /vulnerable/profile/1 (admin's profile)
  3. Capture the profile page showing you can view another user's data

What to capture: The vulnerable profile page showing the IDOR warning and the admin's profile data visible to a regular user.

4. Secure Admin Protection (403)

Description: Demonstrate that the secure implementation properly blocks regular users from accessing the admin panel with a 403 error.

Steps:

  1. Login as regular user (user / user123) in secure demo
  2. Try to navigate directly to /secure/admin
  3. Capture the 403 Access Denied page

What to capture: The 403 Access Denied page showing the secure access control working correctly.

5. Secure Profile Protection

Description: Demonstrate that the secure implementation prevents IDOR by blocking users from viewing other users' profiles.

Steps:

  1. Login as regular user (user / user123) in secure demo
  2. Try to navigate to /secure/profile/1 (admin's profile)
  3. Capture the 403 Access Denied page

What to capture: The 403 Access Denied page showing the IDOR protection working correctly.

6. Secure Admin Access (Authorized)

Description: Demonstrate that an admin user can properly access the admin panel in the secure implementation.

Steps:

  1. Login as admin (admin / admin123) in secure demo
  2. Navigate to /secure/admin
  3. Capture the admin panel showing successful authorized access

What to capture: The secure admin panel showing the admin's username and role, demonstrating proper authorized access.

Screenshot Tips

  • Use your browser's screenshot function or a tool like Snipping Tool
  • Make sure the URL bar is visible to show the route being accessed
  • Capture the full page content, not just a portion
  • Save screenshots in PNG or JPG format
  • Name screenshots descriptively (e.g., vulnerable-admin-access.png)
